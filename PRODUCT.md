# Product

## Register

product

## Users

Один пользователь — SEO-специалист с несколькими Google-аккаунтами, у каждого свой набор Properties в Search Console. Контекст: за рабочим столом на маке, dev-сервер локально, иногда заходит проверить положение по нескольким сайтам сразу. Job-to-be-done: не логиниться вручную в каждый GSC, видеть данные по всем сайтам всех аккаунтов одной поверхностью и быстро забирать выгрузки. Мульти-юзер не предполагается.

## Product Purpose

Локальный self-hosted мульти-аккаунт хаб для Google Search Console. Подключаешь несколько Google-аккаунтов через OAuth (токены хранятся локально в SQLite, ничего больше). Видишь сводку по сайтам, агрегированные ключи и страницы за выбранный период, дашборд карточек с sparkline-графиком clicks по дням и дельтой к предыдущему периоду, по клику на ключ — 16-месячная история позиции. Никакого кеша GSC-данных: всё live-fetch при каждом заходе. Успех = меньше времени уходит на ручные сводки и логин-карусель.

## Brand Personality

Тихий, плотный, без фуфла. Тон — рабочий, без маркетинга и эмодзи. Эмоция — спокойная сосредоточенность дата-аналитика, который доверяет цифрам, а не подсветке. Голос ближе к Stripe Dashboard и Linear, чем к консьюмерскому SaaS. Ноль анимированных градиентов, ноль hero-метрик с огромными числами, ноль декоративных карточек.

## Anti-references

- Salesforce / HubSpot / Zoho-style дашборды: захламлённые виджеты, цветной шум, всё орёт одинаково громко.
- Notion-стилистика на маркетинговых страницах: эмодзи, пастель, скругления 16+ px, иллюстрации с человечками.
- SaaS-кликбейт: hero-метрики «42 318% growth», gradient-text заголовки, glassmorphism cards с blur'ом.
- Любой «AI dashboard» 2024-го: фиолетово-розовый градиент по диагонали, sparkle-иконки, «AI Insights ✨».
- Тяжёлые chart-libs (chart.js, ECharts) с дефолтными темами и тултипами на всём подряд.

## Design Principles

1. **Live, не кеш.** Пользователь нажимает Refresh и реально получает свежие данные из GSC. Никакого хранилища аналитики в БД — только OAuth-токены. Если медленно — медленно честно.
2. **Плотность важнее воздуха.** Информации много, экран широкий. Полная ширина окна, мелкий компактный шрифт, таблицы вместо карточек, минимум отступов между строками. Никаких max-w-7xl ограничителей и центрированных колонок.
3. **URL — это состояние.** Period, sort, dir, cols живут в query-параметрах. Любое представление можно расшарить ссылкой и забукмаркать. Browser-back возвращает прошлое состояние.
4. **Persisted локально, не на сервере.** Что относится только к одному браузеру (скрытые сайты, расширенный ключ) — в localStorage. БД хранит только то, что нельзя восстановить (OAuth refresh-токены).
5. **Стандартные SEO-конвенции.** Position 1 — выше на графике, lower number = better. Не перевирать домен (`sc-domain:` — внутренний формат GSC, для пользователя — чистый домен). Числа форматируются `Intl.NumberFormat('en-US')` с разделителями.

## Accessibility & Inclusion

- WCAG-сертификации не нужно, но базовый contrast и keyboard-nav — норма (Tab по таблицам и формам должен работать без ловушек).
- Цветовые состояния (зелёный рост / красный падение) дублируются знаком (`+12.4%` / `−3.0%`), не только цветом — для дальтоников.
- Reduced motion: тяжёлых анимаций нет, sparkline — статичный SVG. Если когда-то добавятся transitions, через `@media (prefers-reduced-motion: reduce)` отключать.
- Tooltip'ы пока нативные `title=""` — screen-reader-friendly без отдельного `aria-describedby`.
